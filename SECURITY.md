# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please do not disclose it publicly before it has been reviewed.

Instead, contact the maintainers with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested mitigation (if available)

We will acknowledge reports as quickly as possible and work toward a responsible resolution.

---

## Supported Versions

Security fixes are generally provided for the latest maintained version of Hawk.

Older versions may not receive updates.

---

## Scope

Hawk performs read-only Kubernetes dependency analysis and does not intentionally modify cluster resources.

Nevertheless, security issues involving:

- Authentication
- Authorization
- Kubernetes API interactions
- Dependency resolution
- Command execution

should be reported responsibly.
